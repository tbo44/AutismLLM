Goal

Transform the duplicated app into a UK-focused autism facts assistant for autistic people and their carers. It must answer using verifiable facts from trusted UK sources, always cite sources, and avoid individual psychiatric/clinical or legal advice—redirecting users to appropriate services instead.

High-level requirements

Jurisdiction & audience: United Kingdom (with emphasis on England and the London Borough of Hounslow). Audience: autistic people and carers.

Data sources (seed list to crawl/fetch & index):

Government/Statutory: gov.uk (Disability, SEND, EHCP, benefits), legislation.gov.uk (read-only citations), local.gov.uk (as needed), London Borough of Hounslow council pages relevant to autism/SEND/social care.

Health/Clinical: NHS (nhs.uk), NICE guidelines (nice.org.uk) for autism-related guidance and quality standards.

Charities & Advocacy (UK): National Autistic Society, Ambitious about Autism, Autism Alliance UK, Scope, Mencap, IPSEA (education law and SEND), Contact (for families), Carers UK.

Professional/medical associations: Royal College of Psychiatrists, Royal College of Paediatrics and Child Health, Royal College of Speech & Language Therapists (for position statements & facts).

Capabilities:

Retrieval-augmented generation (RAG) over the above sources.

Clear citations (URL + publisher + date) for every non-trivial claim.

Date awareness & recency bias (prefer the latest guidance; show dates).

Hounslow-specific routing (surface local pages/services when user mentions Hounslow or W4/W6/TW postcodes).

Safety/Scope limits:

No diagnosis, treatment plans, medication advice, crisis counseling, or individual legal advice.

Include a standing disclaimer and gentle redirect to GP/NHS 111, emergency services (999), or specialist providers.

For legal questions, provide general information only and suggest contacting IPSEA, Citizens Advice, or a qualified solicitor.

Retrieval pipeline

Add (or update) a simple crawler/fetcher that pulls HTML/PDF from the seed domains above (respect robots.txt; modest concurrency).

Extract clean text, title, canonical URL, publisher, published/updated date, and jurisdiction tags (e.g., country=UK, region=England, local=Hounslow).

Store chunks in a local vector DB (e.g., Chroma/FAISS) plus a lightweight metadata store (SQLite/JSON).

Refresh schedule: weekly for charities/councils; daily for NHS/gov.uk/NICE. (Cron or Replit task.)

When answering, query top-k by semantic similarity + boost recency and authority (NICE/NHS/gov.uk > charity > blogs). Filter to country=UK; if user mentions Hounslow, boost local=Hounslow.

1) Replace system prompt (app-wide)
You are “Compass”—a UK autism facts assistant for autistic people and carers.

MISSION
- Answer with precise, up-to-date facts from trusted UK sources (NHS, NICE, gov.uk, legislation.gov.uk, Hounslow Council, and major UK autism/education/benefits charities).
- Always show citations for substantive facts: [Publisher — Page Title — Date, URL].
- If a question seeks diagnosis, treatment, medication, crisis intervention, or individual legal advice, refuse briefly and offer appropriate next steps.

JURISDICTION & SCOPE
- Prioritise UK law, policy, and services; if location is unspecified, assume UK/England.
- If the user mentions Hounslow or TW postcodes, surface Hounslow-specific services and pages.

STYLE
- Be supportive, plain-English, and concise. Provide step-by-step “what to do next”.
- Summarise first; then add detail; then cite.

GUARDRAILS
- Do NOT provide: diagnoses, personalised therapy, medication dosages, or case-specific legal advice.
- Instead say: “I can’t give personalised clinical/legal advice.” Then guide to: GP/NHS 111; emergency = 999; or named services (e.g., IPSEA for SEND appeals).
- For minors or safeguarding concerns, advise contacting the local authority or NSPCC helpline.
- If sources conflict, state that clearly and present both positions with dates.

CITATIONS
- Cite at least one authoritative source per key claim; include date last reviewed/published if available.
- Prefer the most recent guidance (NICE/NHS/gov.uk). Indicate document dates (e.g., “NICE NG93, updated 2023”).

DATA PRIVACY
- Do not store personal/sensitive data in logs or the vector store. Summaries only. Anonymise examples.

from fastapi import FastAPI
from pydantic import BaseModel
from rag import answerer

app = FastAPI(title="Compass Autism Facts Assistant", version="0.1.0")

class Query(BaseModel):
    question: str

class Answer(BaseModel):
    answer: str

@app.get("/health")
async def health():
    return {"status": "ok"}

@app.post("/chat", response_model=Answer)
async def chat(query: Query):
    text = answerer.answer(query.question)
    return Answer(answer=text)

import chromadb
from chromadb.utils import embedding_functions

_db = None

def get_store(persist_dir: str = "data/chroma_index"):
    global _db
    if _db is None:
        client = chromadb.PersistentClient(path=persist_dir)
        ef = embedding_functions.SentenceTransformerEmbeddingFunction(
            model_name="sentence-transformers/all-MiniLM-L6-v2"
        )
        _db = client.get_or_create_collection(
            name="autism_facts", embedding_function=ef
        )
    return _db

def upsert(documents, metadatas, ids):
    store = get_store()
    store.upsert(documents=documents, metadatas=metadatas, ids=ids)

def query(texts, n_results=3):
    store = get_store()
    return store.query(query_texts=texts, n_results=n_results)

rag/answerer.py
from dataclasses import dataclass
import re
from typing import Optional

@dataclass
class GuardrailTemplates:
    clinical: str = (
        "I can’t provide personalised clinical advice (diagnosis, treatment, or medication changes). "
        "A good next step is to speak to your GP or call NHS 111. "
        "If this is an emergency, call 999."
    )
    legal: str = (
        "I can’t give case-specific legal advice. "
        "For SEND and education issues, you could contact IPSEA. "
        "You can also speak with Citizens Advice or a qualified solicitor."
    )
    crisis: str = (
        "If you’re in crisis or feel unable to stay safe: call 999, go to A&E, or contact NHS urgent mental health help. "
        "You can also call Samaritans on 116 123 (free, 24/7)."
    )

TEMPLATES = GuardrailTemplates()

CLINICAL_PATTERNS = [
    r"\b(dose|dosage|increase|decrease|change)\b.*\b(med|medication)\b",
    r"\b(start|stop|switch)\b.*\b(med|medication)\b",
    r"\b(diagnos(e|is)|treatment plan|therapy plan)\b",
]

LEGAL_PATTERNS = [
    r"\b(win|guarantee|ensure)\b.*\b(tribunal|appeal|case)\b",
    r"\blegal advice\b",
    r"\bhow (do|to) (appeal|win)\b.*",
]

CRISIS_PATTERNS = [
    r"\b(self[-\s]?harm(ing)?|suicid(al|e)|overdose|can.?t cope|in crisis)\b",
    r"\b(unable to stay safe|immediate danger)\b",
]

def _match_any(text: str, patterns) -> Optional[str]:
    for p in patterns:
        if re.search(p, text, re.IGNORECASE | re.MULTILINE):
            return p
    return None

def apply_guardrails(user_text: str) -> Optional[str]:
    if _match_any(user_text, CRISIS_PATTERNS):
        return TEMPLATES.crisis
    if _match_any(user_text, CLINICAL_PATTERNS):
        return TEMPLATES.clinical
    if _match_any(user_text, LEGAL_PATTERNS):
        return TEMPLATES.legal
    return None

def answer(user_text: str) -> str:
    refused = apply_guardrails(user_text)
    if refused:
        return refused
    return "(stub) No guardrails triggered. This is where RAG + LLM synthesis will appear.\n" + user_text
