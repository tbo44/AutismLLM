Prompt for Replit Agent 3 — Keep 400px Focus, Add Optional Expand

Update the MAYA UI to keep the current narrow, mobile-style layout by default on all devices, while providing an accessible toggle that widens the app on larger screens. Persist the user’s choice in localStorage. Do not change backend logic.

Goals

Default Focus Mode: centered column, max width 400–420px, full height, no horizontal scroll.

Optional Expanded Mode: on toggle, widen to 800–960px (cap at min(96vw, 1000px)), still single column (no side panels).

Persist Preference: remember the last mode per device via localStorage("maya-layout").

A11y: keyboard and screen-reader friendly; avoids motion if user prefers reduced motion.

Required changes
A) HTML / React structure

Add a top-right layout toggle button in the header:

Text: “Expand width” ↔ “Narrow width”

aria-pressed reflects state.

Keyboard accessible (Enter/Space).

Add a state class on the root app container: .app[data-layout="focus"] (default) or .app[data-layout="expanded"].

B) CSS

Remove any fixed 400px width rules applied globally; scope width to the main container.

C) JS (vanilla) or React logic

On load:

Read localStorage.getItem("maya-layout"). If not set, default to "focus".

Set document.querySelector(".app").dataset.layout = state.

On toggle:

Switch between "focus" and "expanded".

Update aria-pressed and button label.

Save to localStorage.

Vanilla JS snippet (adapt to React if needed):

(function () {
  const app = document.querySelector('.app');
  const btn = document.querySelector('.layout-toggle');
  const key = 'maya-layout';
  const saved = localStorage.getItem(key);
  const prefers = saved === 'expanded' ? 'expanded' : 'focus';
  app.dataset.layout = prefers;
  btn.setAttribute('aria-pressed', String(prefers === 'expanded'));
  btn.textContent = prefers === 'expanded' ? 'Narrow width' : 'Expand width';

  function toggle() {
    const next = app.dataset.layout === 'expanded' ? 'focus' : 'expanded';
    app.dataset.layout = next;
    localStorage.setItem(key, next);
    btn.setAttribute('aria-pressed', String(next === 'expanded'));
    btn.textContent = next === 'expanded' ? 'Narrow width' : 'Expand width';
  }
  btn.addEventListener('click', toggle);
  btn.addEventListener('keydown', (e) => {
    if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); toggle(); }
  });
})();

Phone (360×640): default = focus; no horizontal scroll; toggle present but keeps 100% width (max 420px).

Tablet (768×1024): default = focus; toggle expands to ~800–960px; readable line lengths.

Laptop/Desktop (≥1366px wide): default = focus (centered column). Toggle expands to ~800–960px and persists after reload.

A11y: toggle reachable by Tab; Space/Enter activate; aria-pressed changes; honors prefers-reduced-motion.

Non-goals

No multi-column/side panels.

No changes to backend endpoints or streaming.
