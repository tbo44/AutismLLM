Goal:
Update the Maya web UI so that each chat message is concise, with sources hidden in a collapsible panel and a persistent mini disclaimer anchored at the bottom of the chat. Keep the existing autism-friendly narrow layout (≈400px max width) across all devices.

Context & Requirements

Design philosophy: Maintain a narrow, low-stimulus reading column (max-width: 400px), centered on larger screens. No multi-column desktop layout.

Message structure (input to renderer):

type Source = {
  title: string;        // e.g., "Access to Work grant"
  url: string;          // absolute https link
  publisher?: string;   // e.g., "Gov.UK" | "NHS" | "NAS"
};

type MayaMessage = {
  role: "assistant" | "user";
  content: string;      // markdown allowed (links, bullets)
  timestampISO?: string;// e.g., "2025-11-06T13:38:00Z"
  sources?: Source[];   // optional; can be empty or undefined
};


Behavior:

Assistant messages render normally, followed by an optional collapsible “Sources” section only if sources?.length > 0.

Collapsible uses native <details><summary>…</summary>…</details> for accessibility. It must be keyboard-navigable and screen-reader friendly.

Persistent mini disclaimer is shown once, fixed at the bottom of the chat viewport (not inside each message). It is always visible, with a link to a full “About & Disclaimers” page/section.

Respect reduced motion and provide high-contrast focus states. Avoid color-only communication.

Copy (default, configurable):

Summary label: 📚 Sources & disclaimer

Footer mini disclaimer text (one line):
Maya provides general guidance only (not medical or legal advice).

Footer link label: Learn more

Optional expanded disclaimer (inside details, below sources):

This response was generated by Maya. Please verify key details with the original sources.
For medical questions: contact your GP or call NHS 111.
For legal/SEND issues: contact IPSEA, Citizens Advice, or a qualified solicitor.
In a crisis: call 999, contact Samaritans (116 123), or go to A&E.

Deliverables

Create or update the following files (TypeScript + React, Vite or Next.js compatible):

src/components/Message.tsx

Renders a single MayaMessage.

Uses a markdown renderer (e.g., react-markdown) with safe link handling (rel="noopener noreferrer" and target="_blank").

If sources present, render:

<details className="sources">
  <summary><strong>📚 Sources &amp; disclaimer</strong></summary>
  <ul className="sources-list">
    {sources.map(s => (
      <li key={s.url}>
        <a href={s.url} target="_blank" rel="noopener noreferrer">
          {s.publisher ? `${s.publisher} – ` : ""}{s.title}
        </a>
      </li>
    ))}
  </ul>
  <div className="disclaimer-expanded">
    {/* expanded disclaimer copy from props/config */}
  </div>
</details>


If timestampISO exists, render a small, de-emphasized timestamp (localised, e.g., 06 Nov 2025, 13:38), but do not clutter above the fold.

src/components/FooterDisclaimer.tsx

Fixed at bottom; full width of the chat column, not the entire screen.

Content:

<div className="footer-disclaimer" role="note" aria-live="polite">
  <span>Maya provides general guidance only (not medical or legal advice).</span>
  <a href="/about" className="footer-link">Learn more</a>
</div>


Ensure it never covers input fields; when overlapping is possible, add bottom padding to the scroll container equal to its height (using CSS variable or a measured height).

src/components/ChatShell.tsx

Wraps the conversation and input.

Centers the column and enforces the narrow width.

Injects bottom padding equal to the footer height.

Implementation Notes

Render markdown safely (no raw HTML injection). Auto-link bare URLs if markdown parser supports it.

Ensure the <details> element starts collapsed by default.

For timestamps, format using the browser’s locale with day-month-year (UK), e.g., 06 Nov 2025, 13:38. Provide a formatTimestamp utility.

All external links open in a new tab and include rel="noopener noreferrer".

Add unit tests for:

Message with no sources: no <details> rendered.

Message with sources: <details> rendered with correct list.

Footer rendered once regardless of message count.

Keyboard accessibility: Tab to summary, Enter/Space toggles.

Example Usage (Message rendering)
import Message from "./components/Message";

const demo: MayaMessage = {
  role: "assistant",
  content: "You can request a Care Act assessment through your local council’s adult social care team.",
  timestampISO: "2025-11-06T13:38:00Z",
  sources: [
    { title: "Care Act assessment for adults", url: "https://www.autism.org.uk/...", publisher: "NAS" },
    { title: "Access to Work grant", url: "https://www.gov.uk/access-to-work", publisher: "Gov.UK" },
    { title: "Autism assessment and diagnosis", url: "https://www.nhs.uk/conditions/autism/getting-diagnosed/assessments/", publisher: "NHS" }
  ]
};

// In the chat list
<div className="message">
  <Message message={demo} expandedDisclaimer={
    <>This response was generated by Maya. Please verify key details with the original sources. For medical questions: contact your GP or call NHS 111. For legal/SEND issues: contact IPSEA, Citizens Advice, or a qualified solicitor. In a crisis: call 999, contact Samaritans (116 123), or go to A&amp;E.</>
  } />
</div>

Acceptance Criteria

 Assistant messages show no bulky lists by default.

 Clicking/pressing the “📚 Sources & disclaimer” summary reveals a list of sources (if any) and the expanded disclaimer text.

 A single persistent footer disclaimer remains visible at the bottom of the chat column without overlapping the composer.

 Layout remains centered, max-width 400px on all devices.

 Fully keyboard accessible; visible focus rings; supports prefers-reduced-motion.

 All external links open safely in a new tab.
